Formats the column using the specified format and the formatting conventions of the current culture. The format values are the same as used in value.ToString(format) but does not really convert the value to a string, just sets the format of the column. Invalid or missing codes default to Excel default format.
See http://www.cheat-sheets.org/saved-copy/msnet-formatting-strings.pdf
See http://blog.stevex.net/string-formatting-in-csharp/

Valid DateTime Format Codes (en-US examples, case-sensitive):

d = ShortDatePattern¹ (ex. 2/4/2020)
D = LongDatePattern¹ (ex. Tuesday, February 4, 2020)
t = ShortTimePattern¹ (ex. 1:34 PM)
T = LongTimePattern¹ (ex. 1:34:26 PM)
f = LongDatePattern¹ + ShortTimePattern¹ aka FullShortDateTimePattern (ex. Tuesday, February 04, 2020 1:34 PM)
F = LongDatePattern¹ + LongTimePattern¹ aka FullLongDateTimePattern (ex. Tuesday, February 04, 2020 1:34:26 PM)
g = ShortDatePattern¹ + ShortTimePattern¹ aka GeneralShortTimePattern (ex. 2/4/2020 1:34 PM)
G = ShortDatePattern¹ + LongTimePattern¹ aka GeneralLongTimePattern (ex. 2/4/2020 01:34:26 PM)
M = MonthDayPattern (ex. February 04)
o = Round-trip date/time pattern² (ex. 2020-02-04T13:34:26.775)
O = Round-trip date/time pattern² (ex. 2020-02-04T13:34:26.775)
r = RFC1123Pattern³ (ex. 04 Feb 2020 13:34:26 GMT)
R = RFC1123Pattern³ (ex. 04 Feb 2020 13:34:26 GMT)
s = SortableDateTimePattern (ex. 2020-02-04T13:34:26)
S = Invalid. Defaults to "g" format.
u = UniversalSortableDateTimePattern (ex. 2020-02-04 13:34Z)
U = UniversalSortableDateTimePattern (ex. 2020-02-04 13:34:26Z)
x = CustomShortDateTimePattern⁴ (ex. 02/04/2020 01:34 PM)
X = CustomLongDateTimePattern⁴ (ex. 02/04/2020 01:34:26 PM)
y = YearMonthPattern (ex. Feb 2020)
Y = YearMonthPattern (ex. Feb 2020)

¹May be overridden by custom format info.
²This pattern cannot be accurately represented in Excel as it does not understand timezone offsets. This is not a localized pattern.
³The values are not automatically converted into GMT. It is up to the caller to convert them first.
⁴Localized pattern contains leading zeros so all values in the column may align.

Valid Number Format Codes (case-insensitive):

f = Fixed point
n = Number with commas for thousands
c = Currency

The numeric format styling code may contain 2 comma delimited fields.
The first field contains the code char + n where the n represents the number of digits after the decimal (aka precision). If the precision is not defined, it defaults to 2.
The optional 2nd field contains the units string(including leading and trailing whitespace). However if the 2nd field contains an integer, it is used as a relative column index to the column containing the units.
Examples: format="f3, mg" will be displayed as "5.320 mg". When precision is undefined, the precision==2. To To force zero digits after the decimal, use "f0".

Valid TimeSpan Format Codes (case-sensitive):

g = low precision e.g. d.hh:mm (default)
G = high precision e.g. d.hh:mm.ss.fff
